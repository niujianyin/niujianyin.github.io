TP5 URL路径格式

url不区分大小写 index tp5默认保留路径
url区分大小写 :config.php url设置 :url_convert false
官方不区分大小写 path_info
不会使用 写接口 最好不写兼容模式和path_info
缺点：太长，url路径暴露服务器文件结构,不够灵活 不能很好的支持Url语义化,路由解决

TP5 路由模式 application/config.php url_route_must:true

XAMPP 下配置虚拟主机

TP5 三种URL访问模式 /Applications/xampp/xamppfiles/htdocs/zerg/application/route.php

$ 选用动态注册(路由模式)； 配置模式(PATH_INFO) 弃用 
$ use think\Route;
$ Route::rule('hello', 'sample/Test/hello');
$ /Applications/xampp/xamppfiles/htdocs/zerg/application/config.php url_route_must: true //强制使用路由模式
$ 定义路由
$ Route::rule('路由表达式'，'路由地址','请求类型','路由参数(参数)','变量规则(数组)')
$ Route::rule('hello','sample/Test/hello','GET|POST',['https'=false]);

GET POST DELETE PUT
请求类型没有 默认会变成 任意请求类型 任意一种类型都支持
路由参数
https 检测是否https请求 不是https 不能访问
Route::any(); 对应

获取参数的方法：
1.传统获取参数，方法中 hello($id,$name)
a.路由中传参 b.?问号方式传参 c.body里传参
2.使用tp5 Request类
Request::instance()->param();获取所有参数。
Request::instance()->route();获取路径里参数。
Request::instance()->get();获取?后面参数。
Request::instance()->post();获取post参数。
依赖注入：hello(Request $request)
3.使用助手函数：input(‘param.id’)
